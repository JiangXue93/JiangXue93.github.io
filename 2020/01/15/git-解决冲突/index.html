git 解决冲突

发表于 2020-01-15 | 分类于 git

今年xxx项目和同事一起组队打怪，好久没有协作啦。git操作真是不熟练，记录下两人修改了同一个文件后，合并文件并提交的步骤。对，就是解决提交代码时候发现有冲突的情况。

之前发布代码用的可视化的界面SourseTree，可视化真是个好东西，只管点点点就行了。但是一不注意有冲突，很大概率把别人提交的给合并错。所以还是回归命令行吧。

得益于vscode的git模块非常方便。下面讨论的处理方法就均在vscode下。

冲突的情况下，拉代码拉不下来，发代码发不出去。怎么办呢，感觉就像是要交换AB两个变量的值（不要用先进的解构），需要引入一个存储的地方。处理git冲突也可以这么干：

  1. 存储当前代码

  2. 拉代码

  3. 把存储的代码恢复

  4. 解决冲突
    在vscode的git面板查看有冲突的文件，挨个解决即可。

  5. 提交代码

附录

  • git stash list 查看stash中有哪些存储
  • git stash show 显示做了哪些改动，默认show第一个存储,如果要显示其他存贮，后面加stash@{$num}，比如第二个 git stash show stash@{1}
  • git stash apply 应用某个存储,但不会把存储从存储列表中删除，默认使用第一个存储,即stash@{0}，如果要使用其他个，git stash apply stash@{$num} ， 比如第二个：git stash apply stash@{1}
  • git stash drop stash@{$num}丢弃stash@{$num}存储，从列表中删除这个存储
  • git stash clear删除所有缓存的stash

经过上面步骤，冲突就解决啦。下班~
