gnvm管理node版本

如何在开发环境切换node版本（windows）

下面开始正文：

我办公环境是windows的系统，所以采用gnvm作为管理工具。gnvm的介绍就不搬运了，网上随便找找就有哈。

刚使用gnvm可能还没node，所以可以使用安装node

  1. 安装node

gnvm install 13.6.0
既可以安装node v13.6.0

  1. 查看已经安装的node版本

gnvm ls

  1. 切换node版本为v11.6.0

gnvm use 11.6.0

  1. 安装对应版本的npm

node和npm的对应关系可以去node以往的版本查看。也可以直接使用gnvm npm global确定npm版本是否对应的，命令行会输出

若不对应的话，继续输入“Y”安装最新的npm即可，最后使用npm -v查看版本是否有更新

npm版本更新失败解决方案

之前安装了node早期版本，npm的默认路径在C:\Users\username\AppData\Roaming下，而每次通过npm install npm@latest -g安装的npm的路径在C:\Program Files\nodejs下，导致npm下载成功，但是版本却更新失败。

处理方式：删除C:\Users\username\AppData\Roaming下的npm文件夹即可。
同样的，默认路径下的node，导致node更新不成功。该问题的解决方式也是删除C:\Users\username\AppData\Roaming下的相关文件。
