umi采坑指南

发表于 2019-06-19 | 分类于 react

由于Ant Design Pro2.0之后将构建方式迁移为了umi（乌米），且项目需要，无奈爬坑ing。下面就遇到的问题不定期更新，大部分是小问题，但是以此作为记录。

配置阶段

  1. umi配置文件.umirc.js 或 config/config.js二选一，前者优先级高
  2. UMI_ENV

导致的问题是，/src/page下的models总是报错，后来才发现这个配置的意思是，启用单数模式的目录，例如让 page 变为约定的文件夹，默认是pages。改成singular: false，src下的page和model、service都改成复数形式就完美解决。

使用阶段

  1. 访问全局常量

  2. 运行时的配置文件src/app.js中添加export function onRouteChange(){}报错Invalid key onRouteChange from plugin，原因是生成配置没包含onRouteChange字段的插件，当时采用的umi版本为2.4.3。
    折中方法是/page/.umi/umi.js文件中修改window.g_plugins.init函数，在validKeys里添加'onRouteChange'字段。但是由于.umi文件每次编辑都会重新生成，so需要每次改了代码还得手动修改.umi文件开发效率大大降低。比较危险的处理方式就是升级umi的版本，umi版本功能变化太大，且并不完全兼容低级版本，但我还是干了！运气真好，项目跑起来也没报错，真是谢天谢地！！！
